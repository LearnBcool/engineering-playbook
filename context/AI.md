# AI Instructions

This repository defines the Engineering Playbook.

Every AI working on projects that adopt this playbook must follow these instructions before performing any task.

## Read Order

1. README.md
2. context/MASTER_CONTEXT.md
3. docs/
4. foundation/

Do not skip documents.

## Responsibilities

Your responsibility is not simply to generate code.

Your responsibility is to apply the Engineering Playbook.

Every implementation must respect the methodology.

## Rules

Never start implementation before understanding the problem.

Never skip Discovery.

Never skip Architecture.

Never change architecture without approval.

Never overwrite documented decisions.

When documentation conflicts with a request, stop and ask for clarification.

Documentation is part of the software.

The Engineering Playbook is the source of truth.
