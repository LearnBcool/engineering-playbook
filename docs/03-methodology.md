# Engineering Methodology

> **Foundation 0.1**

This methodology defines the standard workflow followed by every project developed under the Engineering Playbook.

It provides a repeatable engineering process focused on understanding problems, making informed decisions, delivering quality software, and continuously improving both the product and the engineering team.

---

# Engineering Lifecycle

Every project follows the same lifecycle, regardless of its size or technology.

```
Problem
    │
    ▼
Discovery
    │
    ▼
Research
    │
    ▼
Architecture
    │
    ▼
Planning
    │
    ▼
Development
    │
    ▼
Testing
    │
    ▼
Deployment
    │
    ▼
Reflection
    │
    ▼
Knowledge
```

This workflow is mandatory for every project.

Skipping stages increases technical debt and reduces software quality.

---

# Phase 1 — Discovery

## Objective

Understand the real problem before proposing solutions.

## Activities

- Identify stakeholders.
- Understand business objectives.
- Define user needs.
- Identify constraints.
- Ask questions.
- Avoid assumptions.

## Deliverables

- Problem Statement
- Stakeholder List
- Initial Requirements

---

# Phase 2 — Research

## Objective

Study the problem before making engineering decisions.

## Activities

- Analyze existing solutions.
- Evaluate technologies.
- Study best practices.
- Identify risks.
- Compare alternatives.

## Deliverables

- Research Notes
- Technology Evaluation
- Risk Analysis

---

# Phase 3 — Architecture

## Objective

Design the system before implementation.

## Activities

- Define system components.
- Define communication between services.
- Select architecture patterns.
- Plan scalability.
- Plan maintainability.

## Deliverables

- Architecture Diagram
- Technical Decisions
- Initial System Design

---

# Phase 4 — Planning

## Objective

Transform architecture into executable work.

## Activities

- Break work into tasks.
- Define priorities.
- Estimate effort.
- Organize milestones.
- Create roadmap.

## Deliverables

- Backlog
- Sprint Plan
- Roadmap

---

# Phase 5 — Development

## Objective

Implement the planned solution.

## Activities

- Write clean code.
- Follow engineering standards.
- Review code.
- Document decisions.
- Update technical documentation.

## Deliverables

- Source Code
- Documentation
- Pull Requests

---

# Phase 6 — Testing

## Objective

Validate quality before delivery.

## Activities

- Functional testing.
- Integration testing.
- Performance testing.
- Security review.
- User validation.

## Deliverables

- Test Reports
- Bug List
- Validation Report

---

# Phase 7 — Deployment

## Objective

Deliver software safely.

## Activities

- Prepare production environment.
- Deploy application.
- Validate deployment.
- Monitor system health.

## Deliverables

- Production Release
- Deployment Report

---

# Phase 8 — Reflection

## Objective

Learn from the project.

## Activities

- Review successes.
- Identify mistakes.
- Improve methodology.
- Record lessons learned.

## Deliverables

- Retrospective
- Improvement Actions

---

# Phase 9 — Knowledge

## Objective

Transform experience into reusable engineering knowledge.

## Activities

- Update documentation.
- Update templates.
- Improve playbook.
- Record architectural decisions.

## Deliverables

- Updated Playbook
- New Templates
- Engineering Knowledge

---

# Continuous Improvement

Every completed project must improve at least one of the following:

- Engineering knowledge
- Team methodology
- Documentation
- Templates
- Software quality
- Technical leadership

Projects are temporary.

Knowledge is permanent.
