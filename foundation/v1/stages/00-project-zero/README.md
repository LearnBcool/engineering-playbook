# Stage 00 — Project Zero

> Foundation v1

Project Zero is the official starting point of every engineering project.

No architecture is designed.

No code is written.

No technology is selected.

The only objective is to understand the project before attempting to solve it.

Every project executed using the Engineering Playbook begins here.

---

# Purpose

Transform an idea into a clearly defined engineering problem.

Project Zero aligns stakeholders, establishes the project's purpose, identifies constraints, and defines success before any technical work begins.

Engineering starts with understanding.

---

# Inputs

A project idea.

The idea may be incomplete.

The purpose of Project Zero is to transform uncertainty into clarity.

---

# Participants

Required:

- Engineering Leader
- Problem Analyst
- Research Engineer
- Leadership Observer

Optional:

- Domain Specialist
- Product Owner
- Client Representative

Participants should be limited to those necessary for understanding the problem.

---

# Activities

During Project Zero the team must:

- Understand the purpose of the project.
- Identify the real problem.
- Identify stakeholders.
- Define constraints.
- Define assumptions.
- Identify risks.
- Define success metrics.
- Identify open questions.
- Decide whether the project should continue.

No implementation decisions are made during this stage.

---

# Deliverables

Project Zero must produce:

- Project Charter
- Problem Statement
- Initial Risk List
- Success Metrics
- Open Questions

These documents become the foundation for the next stage.

---

# Definition of Done

Project Zero is complete only when:

- The problem is clearly defined.
- The project purpose is documented.
- Stakeholders are identified.
- Constraints are documented.
- Success metrics are measurable.
- Known risks are recorded.
- Open questions are identified.
- The Engineering Leader approves progression.

Only then may Discovery begin.

# Engineering Notes

Project Zero intentionally avoids discussing technologies.

Choosing technologies before understanding the problem introduces unnecessary bias into the engineering process.

---

# Engineering Principle

Understand first.

Build later.
